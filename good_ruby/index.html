Introduction

This is the second book in the series of programming books authored by me, Jan Savage. This book teaches both intermediate programming and intermediate Ruby: no ruby background is assumed, though. A typical learner would be someone who has finished programming 101 in any language (say, Chris Pine’s Learn to Program because it uses Ruby), or learned some programming here and there–Youtube?–but now wants to consolidate their gains and go to the next level.

Pedagogical model

Or how I teach. Please the introduction in “to Julia” for a deeper discussion. In short, our method is what is known as the spiral learning method: we introduce enough of something to get started, go up a bit, spiral back to the same topic to dig deeper and enhance learning and long-term memorization, and then proceed to the next-tier spiral. We also avoid demotivating learners by introducing things that were not explained, or by confusing learners by poor explanations or errata.

Frequently, I will also mention how Ruby differs from other languages, but no familiarity with the other languages is assumed: sufficient explanations will be provided to ensure maximum comprehension.

Why Ruby?

Because Ruby rocks. Currently, it is one of the best languages, out there. Best for what? Well, for almost any thing. By best, I mean the following:

  • Clear, expressive syntax, programmer-friendly.
  • Superior flexibility, remarkable power. Industrial-grade.
  • Easy to learn, useful for many domains.
  • A true Object-oriented language that is also functional!
  • A forward-looking old language. Big community, tons of gems.
  • Very similar to Crystal and Elixir, if you want to branch out in new, exciting directions.

The Tools

Because this book assumes an advanced beginner level in programming, detailed installation instruction will not be given here. I assume you can install Ruby on your machine, ideally with Homebrew. However, some preferences will be laid out.

First of all, it is preferred that your operating system is either a MacOS or any Unix-based OS (such as any flavor of Linux); if you have Windows, the Linux sub-system on Windows will solve this problem. I use a Mac, so I will speak from this point of view, but everything can be extrapolated to Linux as well.

Next, with regard to the IDE, choose whatever you like. I use and recommend RubyMine. The next best choice is Atom.

Finally, do not use your pre-installed Ruby (the system Ruby). Use Homebrew to install the latest Ruby version. All the code in this book works with Ruby 3 and up.

We will now directly proceed.

Everything is an Object

It is assumed that you know what are int, floats, strings, booleans, nil or null, variables, assignment, lists, dictionaries, iteration, recursion, tuples, loops, conditionals, basic object-oriented concepts such as objects, instances, classes, methods; functions, nested conditionals, nested loops, and ternary operators. If you are rusy, please proceed to Julia, which is a programming 101 for absolute beginners or those who need to brush up on their knowledge.

Now then, in Ruby, variables are disloyal:

thing = 34
thing = "what?"
thing = true 
thing = 3.2
thing = [1, 4, 4]
thing = thing.nil?

In other words, our variable thing can assume any data type. First it became an int, then a string, then a boolean, a float, a list (called array in Ruby) and finally, a result of evaluating whether the thing object is a nil or not? In this case, thing is assigned a new boolean value (false).

This need not be cause for alarm. Ruby also has what are called constants. A constant is an assignment whose value never changes. By the way, constants are always written in capital letters.

STUBBORN = "you can't change me. Yohoo!"

By contrast, variables are written in small letters and if there are more words, then a snake_case is used:

this_is_it = "I became a snake"
your_birthyear = 1844

In Ruby, everything is an object. Everything:

The “class” of an object determines what type of an object it is. A dog is of class animal, for instance. Any dog object, is the animal class, therefore. But classes can be fine-tuned. A dog can also be a canine class (but not a feline class), although both are the animal class. More on this later.
